==========================================
BANK-API DEPLOYMENT SOLUTIONS
==========================================

PROBLEM:
The "Unable to find fallback package folder" error is a SYSTEM-LEVEL issue, not project code.
It's caused by your .NET SDK installation registering a missing fallback path.
This only affects LOCAL builds, NOT Azure deployments.

==========================================
SOLUTION 1: USE AZURE DEPLOYMENT (RECOMMENDED)
==========================================

Azure has a CLEAN environment with proper NuGet configuration.
The project will build successfully on Azure.

Steps:
1. Push code to GitHub (already done)
2. Go to Azure Portal → Create Web App
3. Select "GitHub" as deployment source
4. Authorize GitHub
5. Select Bank-Api repository and main branch
6. Azure will automatically:
   - Build the project (in clean environment)
   - Run tests
   - Deploy to App Service
   - Apply database migrations

No local build needed!

==========================================
SOLUTION 2: USE DOCKER BUILD (LOCAL)
==========================================

Docker provides a clean build environment like Azure.

Prerequisites:
- Docker Desktop installed

Steps:
1. Open PowerShell in project root
2. Build Docker image:
   docker build -t bank-api:latest -f Dockerfile.prod .

3. Run locally to test:
   docker run -p 8080:80 -e ConnectionStrings__DefaultConnection="Server=db48070.public.databaseasp.net;Database=db48070;User Id=db48070;Password=8s@A=4FaZ-k6;Encrypt=True;TrustServerCertificate=True;MultipleActiveResultSets=True;" bank-api:latest

4. Test API: http://localhost:8080/swagger

==========================================
SOLUTION 3: UNINSTALL/REINSTALL .NET SDK
==========================================

If you need to work locally:

Steps:
1. Uninstall: dotnet --version (note version)
2. Uninstall all .NET SDKs from Control Panel → Programs
3. Delete folder: C:\Users\[YourName]\.dotnet
4. Delete folder: C:\Program Files\dotnet
5. Restart computer
6. Download and install .NET 9.0 SDK from: https://dotnet.microsoft.com/download/dotnet/9.0
7. Restart again
8. Run: dotnet restore

==========================================
SOLUTION 4: FIX VIA ENVIRONMENT VARIABLE
==========================================

This prevents fallback folder resolution:

PowerShell (temporary):
$env:DOTNET_SKIP_FIRST_TIME_EXPERIENCE="true"
$env:DOTNET_SKIP_FIRST_TIME_EXPERIENCE="true"
dotnet restore

Windows Environment Variable (permanent):
1. Win + X → System
2. Advanced system settings → Environment Variables
3. New User Variable:
   Name: DOTNET_SKIP_FIRST_TIME_EXPERIENCE
   Value: true
4. Restart PowerShell
5. Run: dotnet restore

==========================================
SOLUTION 5: USE .NET CLI WITH CUSTOM SOURCES
==========================================

Force NuGet to use only official source:

PowerShell:
cd src
dotnet restore --source https://api.nuget.org/v3/index.json
dotnet build
dotnet publish -c Release -o ./publish
